liteide安装gdb

发表于 2017-03-17 | 分类于 go | 阅读次数

学习go有一段时间了一开始选择 liteide 做 golang 开发工具，在这里记录一下怎么进行debug调试
liteide的debug调试是依赖于gdb工具

步骤

安装gdb

配置gdb路径

liteide中点击[编辑当前环境]会打开system.env文件，将LITEIDE_GDB改成你的地址

system.env
#native compiler drawin amd64
#GOROOT=/usr/local/go
#GOBIN=
#GOARCH=amd64
#GOOS=darwin
#CGO_ENABLED=1
#PATH=$GOROOT/bin:$PATH
LITEIDE_GDB=/usr/local/bin/gdb
LITEIDE_MAKE=make
LITEIDE_TERM=/usr/bin/open
LITEIDE_TERMARGS=-a Terminal
LITEIDE_EXEC=/usr/X11R6/bin/xterm
LITEIDE_EXECOPT=-e

macos系统无法调试

我在升级macos系统之后发现即使gdb配置正确也无法正常调试，应该是跟系统冲突了。最后找到的解决方法是在需要调试的源码当前目录下创建一个.gdbinit文件, 加入一行:

set startup-with-shell off
这个方法麻烦在每个源码目录下都得放置一个 .gdbbinit文件
